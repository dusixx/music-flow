## Что было сделано:

### 1. Роутинг

Настроил роутинг для трех маршрутов: `login`, `signup`, `**`. Для первых двух настроил `canActivate` (`guestGuard`), перенаправляющий на главную страницу в случае, если пользователь уже авторизован (возрат `UrlTree`). Для всех маршуртов настроил `lazy` загрузку. В guestGuard инжектировал `AuthSevice` для проверки статуса аутентификации. Сам AuthService пока "мокнутый". Когда подъедет полноценная реализация перепишу гард. Никаких проблем в процессе не возникло.

**Затраченное время: 2 часа**

### 2. Main layout

Добавил `sidebar` со ссылками на основные страницы приложения и возможностью добавить дополнительный контент. Например, информацию о текущем треке. Сделал боковую панель `resizable`. Для этого использовал готовое решение `angular-split`. Панель можно свернуть/развернуть двойным щелчком по сплиттеру между основным контентом и собственно панелью.

Очевидно, что такой лейаут подходит больше для десктопа. Потому было принято решение реализовать еще и мобильный лейаут и переключаться между ними в главном по заданным брейкпоинтам.

Для этого реализовал простой `BreakpointService`, отслеживающий основные точки перелома (`mobile | tablet | desktop`). При ширине экрана меньше чем `tablet` - sidebar схлопывается в минимальный размер. При ширине менее `mobile` - идет смена лейаута на мобильный, в котором sidebar отсуствует в принципе. Чтобы обеспечить навигацию в этом случае - добавил мобильное меню, выезжающее справа. Так же добавил поле для поиска в хедер.

**Затраченное время: 48+ часов**

## Проблемы и решения:

Оказалось, что при коммитах неверно отрабатывал `lint-staged`. В конфиге были прописаны скрипты из `package.json`
```
*.{ts,js}': ['npm run lint:fix', 'npm run format:fix']
...
``` 
Похоже, список `staged` файлов не передавался в них и каждый раз проверялись файлы всего проекта. Заменил скрипты на комманды (`eslint --fix`, `prettier --write ...`) и все заработало. Вероятно, можно было добавить `--` для скриптов (`npm run lint:fix --`). И это тоже помогло бы.

При клике на кнопку кебаб-меню и последующем закрытии меню нажатием `Esc`. Вокруг кнопки появлялась фокусная рамка. Чтобы избежать этого при клике сбрасываю фокус (`blur()`).

## Что узнал:

1. Очень удобная штука `model()`. Позволяет реализовать двухсторонне связывание `banana-in-box`. Использовал в контексте мобильного меню:

<details>
<summary><b>Details</b></summary>

В хедере есть кнопка меню

```html
<button (click)="menuOpen.set(true)"></button>
```

```ts
menuOpen = model(false);
```

Мобильное меню открывается, если `open() === true`

```html
<aside class="drawer" [class.drawer--active]="open()">...</aside>
```

```ts
open = model(false);
```

В мобильном лейауте связываем логику хедера и меню

```html
<player-header class="header" [(menuOpen)]="menuOpen" />
<player-mobile-menu [(open)]="menuOpen" />
<main class="main">
  <router-outlet />
</main>
```

```ts
protected menuOpen = model(false);
```

Таким образом, нажатие на кнопку меню установит `menuOpen` лейаута в `true`. А `open` мобильного меню получит `menuOpen` из лейаута. Для элементов формы существует нативное решение через ngModel, например, `[(ngModel)]="searchQuery"`

</details>

2. Можно получить ссылку на дочерний элемент

<details>
<summary><b>Details</b></summary>

```
<button #menuButton (click)="openMenu()">...</button>
```

`viewChild` вернет signal. То бишь, как только элемент появится в DOM, `menuButton()` начнет возвращать его, а не `undefined`. Прочитать этот сигнал можно, например, в `afetrRenderEffect`. Можно так же указать `required`, если мы точно уверены, что элемент присутствует. В случае с мобильным меню - в обработчике клика кнопки мы точно уверены, что она есть.

```ts
private menuButton = viewChild.required<ElementRef<HTMLButtonElement>>('menuButton');

openMenu() {
  this.menuOpen.set(true);
  this.menuButton().nativeElement.blur();
}
```

</details>

```html
  Login
</button>
```

</details>

4. Можно передать в директиву какие-то параметры, используя фактически ее селектор

<details>
<summary><b>Details</b></summary>

```html
<button routerLinkActive="active" routerLink="/login">Login</button>
```

Чтобы реализовать аналогичное поведение для своей директивы надо указать `exportAs` при настройке. Тогда сможем получать ссылку в шаблоне. И объявить одноименный инпут - тогда сможем передать через селектор директивы данные.

```ts
@Directive({
  selector: 'button[playerButton]',
  exportAs: 'playerButton',
  ...
})
export class Button {
  playerButton = input('');
  // тут можно эффект с чтением playerButton ...
```

```html
<button #menuButton playerButton="some-string" #pb="playerButton"></button>
```

Можно передавать не только строки (как и для любого инпута)

```ts
playerButton = input<string | Record<string, unknown>>();
```

```html
<button [playerButton]="{...}">...</button>
```

</details>

---

5. Можно установить обработчик события через свойство `host: {...}` при настройке компонента/директивы. В примере ниже, закрываем меню по нажатию `ESC`. Делаем это через привязку в host, а не прямой установкой/удалением обработчика в коде MobileMenu

<details>
<summary><b>Details</b></summary>

```ts
@Component({
  ...
  host: {
    '(document:keydown.escape)': 'open.set(false)',
  },
})
export class MobileMenu {
  open = model(false);
}
```

</details>
