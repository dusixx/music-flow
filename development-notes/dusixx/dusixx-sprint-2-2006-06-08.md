### 1. Карточка трека

Создал карточку трека. Над обложкой альбома при наведении появлется оверлей  с кнопками и, вероятно, в перспективе дополнительной информацией (`plays count`, `duration`). Пока вся информация расположена под обложкой. Была идея создать карточку в разных вариантах и ориентациях (вертикальная, горизонтальная). 

**Горизонтальная** - это скорее лейаут для табличной строки. Где от самой карточки остается только обложка с оверлеем (кнопкой `Play`), название трека и артиста. И тут же могут быть различные элементы (кнопки, чекбоксы, доп информация). В зависимости от контекста использования такого горизонтального лейаута. 

Прикинув как все это стилизовать и настраивать через инпуты компонента, от идеи с ориентациями отказался. Сделал отдельный компонент для обложки с оверлеем. И отдельно надо делать горизонтальную "карточку". Для устройств без поддержки ховера - контент оверлея не прячется.

**Обложка** занимает чуть больше половины карточки (в текущей реализации). Поэтому, если навести не на обложку непосредственно, оверлей c `Play` не отобразится. Это может смутить пользователя (UX). Для того, чтобы оверлей появлялся при наведении в любом месте карточки - добавил `isHovered` инпут обложке. 

Прокидываю из карточки `isHovered` обложке при `(mouseenter) | (mouseleave)` только для устройств с поддержкой `hover`. Чтобы отслеживать такие устройства - добавил сигнал `canHover()` в сервис `BreakpointService`. Сам сервис переименовал в `ViewportService`.

Для hover-устройств идникатором того, что трек играет будет сама кнопка `Play`. Когда карточка в состоянии `isPlaying` оверлей с кнопкой не исчезает, а сама кнопка становится `Pause`. Состояние `isPlaying` ставится для карточки извне. 

Предварительно должна произойти инициализация плеера/проверка наличия трека и тп. Карточка лишь сообщает, когда произошло нажатие на кнопку. Для этого у нее (и у обложки) есть события 
```ts
playClicked = output<Track>();
favoriteClicked = output<Track>();
```

Клик по имени трека ведет на страницу альбома, частью которого является трек. Клик по имени артиста - ведет на страницу артиста (`routerLink`).

Для отображения `plays count` в компактном виде и `duration` в формате `mm:ss` реализовал соотвествующие `pipe`. Учитывая, что контейнер для карточки тоже может быть кликабельным (например, ведет на страницу альбома). Сделал директиву `playerStopPropagation`. Которую можно накидывать на любой кликабельный элемент. Использую его для кнопок в оверлее обложки.

### 2. Карусель

После безуспешных попыток "завести" библиотечную карусель решил сделать свою. Во-первых для библиотечной карусели надо было разбивать список на страницы. Во-вторых, похоже, это в приницпе не совсем то решение, на которое я расчитывал.

Оказалось, что проще реализовать свою карусель с нативной поддержкой скролла на любых устройствах. 

**Идея простая:** через стили делаем контейнер `scrollable` (скроллбар скрываем) и ставим свойство `scroll-snap-type: x mandatory;` (ось прокрутки и залипание на отмеченных элементах). Элементы на которых прокрутка обязана остановиться - отмечаем свойством `scroll-snap-align`. 

При вызове методов `prev()` и `next()` контейнер скролится на всю видимую ширину `clienWidth` методом `scrollBy()`. Для контроля крайних положений скрола ставлю `resize observer`. Чтобы при изменении ширины вьюпорта пересчитывать их. Так же пересчет ведется на событии `scroll`. При пересчете ставятся булевые сигналы 
```ts
atStart = signal(true);
atEnd = signal(false);
``` 
Такое остлеживание нужно в основном для того, чтобы тоглить `disabled` на контролах прокрутки карусели.

**Для элемента**, который нужно поменить как `snap` создал простую директиву. Которая накидывает инлайн-стиль `scroll-snap-align: start;` и переданный через одноименное (как селектор директивы) свойство дополнительные классы для стилизации.

В итоге карусель создается так:
```html
<div class="controls">
  <button      
    playerButton
    icon="chevron-left" 
    (click)="carousel.prev()"
    [disabled]="carousel.atStart()">
  </button>
  <button 
    playerButton
    icon="chevron-right" 
    (click)="carousel.next()" 
    [disabled]="carousel.atEnd()">
  </button>
</div>
<player-carousel #carousel class="carousel">
  @for (item of items(); track item.id) {
    <div playerCarouselItem="carousel-item">
      <!-- some content -->
    </div>
  }
</player-carousel>
```

### 4. Стилизация

Создал sass-фунцкию `rem()` для перевода `px` в `rem` по заданной базе. Добавил sass-переменные для отступов, размеров шрифтов и других элементов. На основе этих переменных сделал токены (`--space-`, `--text-`, `--size-`, `--radius-`). 

Заменил на них все `hardcoded` значения в стилях. Все `px` перевел в `rem`. Для случаев, когда не используются токены - можно переводить px в rem функцией `rem()`. 

Единственное, значения должны быть получены в момент компиляции. То бишь, нельзя присвоить css-переменной значение из sass-функции. Надо предварительно сохранить его в sass-переменную.
```scss
@use 'utils/funcs' as *;

.card {
  $width: rem(200);
  --card-width: $width;
}
// или через интерполяцию
.card-2 {
  --card-width: #{rem(150)};
}
// Для обычных свойств все работает непосредственно
.wrapper {
  width: rem(1200);
}
```
